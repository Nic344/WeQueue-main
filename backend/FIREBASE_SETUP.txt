WeQueue Firebase Backend Setup
==============================

1. Create a Firebase project at https://console.firebase.google.com
2. Add an Android app with package name: com.example.queueapp
3. Download google-services.json and replace: app/google-services.json
4. Enable Authentication > Sign-in method > Email/Password
5. Create Firestore database (production or test mode)
6. Deploy security rules from backend/firestore.rules
7. Deploy composite index from backend/firestore.indexes.json (or create when prompted)

Firestore collections (auto-created by the app):
- queue_global/state  -> nowServing, nextCounter
- users/{uid}         -> displayName, email, favoriteIds, activeQueueId
- users/{uid}/history -> queue history entries
- queues/{queueId}    -> active queue tickets

Food catalog remains local (MockDataProvider) for UI assets (drawables/colors).

User roles (field "role" on users/{uid}):
- customer  -> Customer app (MainActivity)
- staff     -> Staff dashboard (StaffMainActivity) — operations, overview, reports, team

Set role manually in Firestore Console for staff test accounts.
New registrations default to role: customer.
Legacy role value "manager" is treated as staff by the app.
